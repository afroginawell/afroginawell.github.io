# vuepress-theme-vdoing本地搭建及相应插件配置

提示

vuepress-theme-vdoing是vuepress的一款主题，想了解具体内容可以浏览vuepress-theme-vdoing官网 (opens new window)

本文是对于vuepress-theme-vdoing本地搭建及相关插件配置的详细说明，是对官网的介绍的进一步说明。有哪些插件可以参考文章目录，如果需要其他插件配置说明，可以在文章下方评论。

# 1. github拷贝vuepress-theme-vdoing

通过git进行拷贝

如果安装了git的，可以通过以下的git命令直接拷贝文件，拷贝的文件在电脑的默认路劲下，一般在C盘的用户的个人文件夹下，如果想安装git，可参考本地git安装

git clone https://github.com/xugaoyi/vuepress-theme-vdoing.git
1

百度搜索“vuepress-theme-vdoing”，进入官网，可以找到对应的github仓库的网址

或者直接跳转vuepress-theme-vdoing的github仓库 (opens new window)，将仓库内所有内容下载到本地，github官网提供了下载整个仓库文件为压缩包的服务。

# 2.安装node.js

vuepress-theme-vdoing需要v14.17.x版本以上的node.js的支持

具体的安装过程可以参考本地node.js安装

# 3.下载相关依赖

注意

如果执行了本地node.js安装的配置全局安装的模拟路径和缓存路径，那么就需要先用管理员权限打开控制命令符面板，方法：win+S打开搜索，输入cmd，会出现“用管理员权限开发”的选项，然后再将命令路径改为新下载的vuepress-theme-vdoing文件夹的路径，输入“npm install”下载相关的依赖，如果警告版本问题，可以使用“npm update”进行更新

ctrl+R，输入cmd进入控制命令符面板，然后将命令路径改为新下载的vuepress-theme-vdoing文件夹的路径

输入“npm install”下载相关的依赖

# 4.测试vuepress-theme-vdoing主题

注意

同理，如果执行了本地node.js安装的配置全局安装的模拟路径和缓存路径，那么就需要用管理员权限打开控制命令符面板

如果没有用管理员权限打开的话，面板会出现logfile警告，无法向npm日志中写入log。npm日志的位置在node_cache文件里，有些时候需要管理员权限才能修改，面板缺少管理员权限，无法向npm日志中写入log，自然会报错。如果不在意这个的话，可以不用管理员权限打开控制命令符面板，运行”npm run dev“。

“npm install”运行完后，相同路径下输入“npm run dev”进行编译，编译完成会出现localhost:8080的提示，在浏览器中打开localhost:8080即可看到官方给出vuepress-theme-vdoing主题测试内容。（建议使用谷歌浏览器打开）

3）根据需求修改 vuepress-theme-vdoing的config.js (opens new window) 配置

4）修改 vuepress-theme-vdoing的首页配置 (opens new window)

# 6.部分插件配置

详细的参考官方的vuepress-theme-vdoing插件配置 (opens new window)，这里只给出几个博主用过的

# 1）代码块一键复制

参考vuepress-plugin-one-click-copy文档 (opens new window)

  • 下载插件依赖

    npm i vuepress-plugin-one-click-copy -D
    1

  • 添加到网站里

    在config.js中添加

    module.exports = {
  plugins: ['one-click-copy', {
    copySelector: ['div[class*="language-"] pre', 'div[class*="aside-code"] aside'], // String or Array
    copyMessage: 'Copied successfully!', // default is 'Copied successfully!'
    toolTipMessage: 'Copy to clipboard', // default is ''Copy to clipboard'
    duration: 300, // prompt message display time
  }]
}

# 2）站点信息模块

参考站点信息模块文档 (opens new window)

文档内容已经非常详细，文档作者非常负责，这里就不再做说明

运行时可能出现的错误：

  • 站点信息模块乱码，可以尝试将自己创建的js和vue文件的编码转换为utf-8，在重新运行
  • 站点最后活动时间和访问次数/排名出现错误，可能是因为本地配置，未上传到github或其他服务器上，可以将文件上传到github或其他服务器上，再查看效果。
